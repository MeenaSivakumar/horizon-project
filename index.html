Relative CSS Units

em

What it means: Relative to the font size of the element’s parent.
Think: “How big am I compared to my parent’s font?”
Example: If the parent’s font-size is 16px, then 1.5em = 24px.
Common uses: Font sizing, spacing that should scale with text.

rem

What it means: Relative to the font size of the root element (html).
Think: “I always compare myself to the main boss — the HTML font size.”
Example: If html { font-size: 16px; } then 1rem = 16px everywhere.
Common uses: Consistent typography, spacing in design systems.

%

What it means: Relative to another property’s value — depends on context.
Think: “I copy a percentage of my parent’s size, but which size depends on the property.”
Example: width: 50% → half the parent’s width. padding-top: 50% → half the element’s width .
Common uses: Responsive widths, flexible padding/margins.

vw

What it means: Relative to 1% of the browser’s width.
Think: “I resize when the browser window changes width.”
Example: On a 1000px-wide screen, 10vw = 100px. Common uses: Full-width layouts, responsive typography.

vh

What it means: Relative to 1% of the browser’s height.
Think: “I resize when the browser window changes height.”
Example: On an 800px-tall screen, 50vh = 400px. Common uses: Hero sections, full-screen layouts.

Media Queries Mobile First Approach

Media queries let you apply CSS only when the screen meets certain conditions.
Mobile-first means: Write your default CSS for mobile screens first.
Use @media (min-width: …) to add styles for larger screens (tablet, desktop).

Common mobile-first breakpoints:

  • Small: 0–480px → phones
  • Medium: 481–768px → large phones & small tablets
  • Large: 769–1024px → tablets & small laptops
  • Extra Large: 1025px+ → desktops
  • Mobile-First Order Base (mobile)
  • Tablet (min-width: 768px)
  • Small desktop (min-width: 1024px)
  • Large desktop (min-width: 1440px)

    • WCAG AAA Contrast Ratio

    Accessibility ensures your site can be used by all users, including those with disabilities. Contrast ratios help users with low vision or in poor lighting.
    WCAG standards:

  • AA: Regular text ≥ 4.5:1
  • Large text ≥ 3:1.
  • AAA: Regular text ≥ 7:1,
  • Large text ≥ 4.5:1.

    • “Large text” means ≥ 18px regular weight, or ≥ 14px bold.
